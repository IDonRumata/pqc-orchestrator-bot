# PQC Strategic Orchestrator Bot

Асинхронный мультиагентный Telegram-бот для стратегии, комплаенса и финансов
стартапа в сфере постквантовой криптографии и кибербезопасности (ЕС, Польша).
Для личного использования.

## Возможности
- Умный роутинг: подключает 1-3 релевантных AI-агента на запрос.
- Агенты: Финансовый директор, Ученый-криптограф, Юрист по комплаенсу ЕС,
  Эксперт по грантам ЕС и Польши, Главный Критик.
- Локальная база знаний (RAG) на PostgreSQL + pgvector.
- Гибридные эмбеддинги: бесплатный fastembed локально + OpenAI как опция.
- Фоновый мониторинг законов и грантов с уведомлениями о критических изменениях.
- Отказоустойчивость: retry, таймауты, graceful shutdown, структурные логи.

## Требования
- Docker и Docker Compose (рекомендуемый способ), либо Python 3.11+ и PostgreSQL 15+
  с расширением pgvector.
- Токен Telegram-бота от @BotFather.
- Ключ OpenRouter. Ключ OpenAI - опционально (для качественных эмбеддингов).

## Быстрый старт через Docker
1. Скопируй пример конфигурации и заполни значения:
   ```bash
   cp .env.example .env
   ```
   Обязательно укажи `TELEGRAM_BOT_TOKEN`, `ALLOWED_USER_IDS`, `OPENROUTER_API_KEY`
   и надёжный `POSTGRES_PASSWORD`.

2. Запусти стек:
   ```bash
   docker compose up -d --build
   ```

3. Посмотри логи:
   ```bash
   docker compose logs -f bot
   ```

4. Наполни базу знаний документами (законы, гранты). Положи файлы в папку
   `knowledge_base/` и запусти ингест внутри контейнера:
   ```bash
   docker compose exec bot python -m src.ingest --path knowledge_base --source legal
   ```

## Запуск без Docker
1. Создай окружение и установи зависимости:
   ```bash
   uv venv
   .venv\Scripts\activate
   uv pip install -r requirements.txt
   ```
2. Подними PostgreSQL с pgvector и пропиши `DATABASE_URL` в `.env`, например:
   ```
   DATABASE_URL=postgresql+asyncpg://pqc:password@127.0.0.1:5432/pqc_orchestrator
   ```
3. Наполни базу знаний:
   ```bash
   uv run python -m src.ingest --path knowledge_base --source legal
   ```
4. Запусти бота:
   ```bash
   uv run python -m src.bot
   ```

## Конфигурация моделей
Все ID моделей задаются в `.env` (см. `.env.example`) и читаются `src/config.py`.
Значения по умолчанию рассчитаны на каталог OpenRouter 2026 года. Перед запуском
сверь слаги моделей с актуальным каталогом OpenRouter и при необходимости поменяй
в `.env` без правки кода.

## Как пользоваться
Напиши боту обычным текстом задачу или мысль. Система сама выберет нужных
экспертов, при необходимости поднимет тексты законов и грантов из базы знаний и,
если экспертов несколько, прогонит ответ через Главного Критика. Фоновый
мониторинг сам пришлёт уведомление о критических изменениях.

Команды: `/start`, `/help`.

## Безопасность
- Реальный `.env` никогда не коммить, в репозитории только `.env.example`.
- База данных слушает только `127.0.0.1`.
- Доступ к боту ограничен белым списком Telegram id, пустой список закрывает
  доступ всем.
