# BRIEF - PQC Strategic Orchestrator Bot

## Назначение
Асинхронный мультиагентный Telegram-бот для личного использования. Помогает
вести стратегию, комплаенс и финансы стартапа в сфере постквантовой
криптографии (PQC) и кибербезопасности в ЕС и Польше. Агрегирует ответы
узкоспециализированных AI-агентов, опирается на локальную базу знаний (RAG)
и оптимизирует расходы на LLM через OpenRouter.

## Архитектура
- `src/config.py` - вся конфигурация через pydantic-settings, ID моделей в ENV.
- `src/logging_config.py` - структурированные JSON-логи.
- `src/models.py` - ORM-модели (knowledge_chunks, monitored_items, agent_runs).
- `src/database.py` - async-движок SQLAlchemy 2.0, pgvector, поиск и запись.
- `src/embeddings.py` - гибрид эмбеддингов: fastembed (локально) + OpenAI,
  оба урезаны до 384 измерений, выбор лучшего провайдера на запрос с фолбэком.
- `src/openrouter_client.py` - async httpx клиент OpenRouter, Pydantic v2, tenacity.
- `src/prompts.py` - изолированные системные промпты всех агентов.
- `src/orchestrator.py` - роутинг, RAG, параллельный запуск агентов, Критик.
- `src/news_monitor.py` - двухслойный фоновый мониторинг законов и грантов.
- `src/bot.py` - aiogram 3.x, access control, typing, graceful shutdown.
- `src/ingest.py` - CLI наполнения базы знаний (txt, md, pdf).
- `src/text_utils.py` - чанкинг с перекрытием.

## Поток обработки запроса
1. Роутер (дешёвая модель) выбирает 1-3 релевантных агента и поисковый запрос.
2. Если выбран legal или grants - косинусный поиск в pgvector поднимает чанки.
3. Выбранные агенты работают параллельно, каждый в своём системном промпте.
4. Если агентов больше одного - Главный Критик сводит и валидирует ответы.
   При одном агенте его ответ отдаётся напрямую (экономия токенов и времени).

## Гибрид эмбеддингов (важно)
Оба провайдера дают вектор размерности 384 (OpenAI через параметр dimensions).
При ингесте считаются оба вектора в отдельные колонки. На запросе выбирается
провайдер по длине запроса, при сбое OpenAI - фолбэк на локальный. Поиск по
openai-колонке при отсутствии данных откатывается на local-колонку.

## Мониторинг (двухслойный)
- Слой 1: цикл раз в 12 часов опрашивает RSS (EUR-Lex, ENISA, NIST, NCBR),
  сравнивает с БД, скачивает новое.
- Слой 2: быстрая модель классифицирует влияние. Критичное - перевекторизуется
  в базу знаний, пользователю уходит уведомление в Telegram.

## Безопасность
- Секреты только в .env (в .gitignore), в репо только .env.example.
- Postgres слушает только 127.0.0.1, пароль через ${ENV} в docker-compose.
- Whitelist пользователей по Telegram id, пустой список = доступ закрыт всем.
- Все HTTP-запросы с таймаутами и retry (tenacity).
- Бот работает под non-root пользователем в контейнере.

## Known issues / на что смотреть
- RSS-эндпоинты могут меняться, список источников в config.py (news_sources).
- ID моделей OpenRouter сверять с актуальным каталогом OpenRouter перед запуском.
- IVFFlat-индекс эффективен после наполнения базы, при малом числе чанков поиск
  всё равно корректен (просто перебором).
- База знаний пустая до первого запуска src/ingest.py.
